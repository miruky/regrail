**正規表現を鉄道図として描き、テスト文字列との照合を1ステップずつ追えるデバッガです。**

## 概要

正規表現を入力すると、その構造を鉄道図(railroad diagram)としてSVGで描画します。テスト文字列を与えると一致範囲と捕捉グループを表示し、「進む」ボタンで照合エンジンの試行・文字の消費・不一致・成功を1ステップずつ再生できます。各ステップでは図中の対応ノードと文字列中の現在位置が同時にハイライトされ、どこで何が起きているかを目で追えます。

正規表現エンジンは自前のバックトラッキング実装で、照合の内部過程をそのままトレースとして取り出せるのが要です。パーサ・図の描画・照合のいずれも外部ライブラリに依存しません。

遊ぶ: https://miruky.github.io/regrail/

### なぜ作ったのか

正規表現は「書けたけれど、なぜマッチする(しない)のか」が見えにくい道具です。とくに量指定子の貪欲・非貪欲やバックトラッキングは、結果だけ見ても挙動が想像しづらく、デバッグが勘頼りになりがちです。構造を図で俯瞰しつつ、エンジンが実際にどう走るかを一歩ずつ観察できれば、正規表現は「読める」ものになります。それを狙って作りました。

## 使い方

正規表現とテスト文字列を入力すると、鉄道図・一致結果・捕捉グループが即座に更新されます。「例」から代表的なパターンを読み込んで試せるほか、入力内容はURLに保存されるので「リンクをコピー」で今の状態をそのまま共有・ブックマークできます。表示テーマはライト・ダーク・自動(OS追従)を切り替えられます。

ステップ操作では、図中の対応ノードとテスト文字列の現在位置が同時にハイライトされ、状態が色で示されます。

- 試行 — そのノードでの照合を試している
- 消費 — 文字を1つ消費して先へ進んだ
- 不一致 — そのノードで一致せず、後戻り(バックトラック)する
- 成功 — パターン全体が一致した

### 対応する記法

| 記法              | 例                                       |
| :---------------- | :--------------------------------------- |
| 文字・任意の1文字 | `a`, `.`                                 |
| 文字クラス        | `[a-z]`, `[^0-9]`                        |
| 短縮クラス        | `\d` `\w` `\s`(および大文字の否定形)     |
| アンカー          | `^` `$` `\b` `\B`                        |
| グループ          | `(...)`, `(?:...)`                       |
| 選択              | `a` と `b` のいずれか(縦棒で区切る)      |
| 量指定子          | `*` `+` `?` `{m,n}`(末尾に `?` で非貪欲) |

## アーキテクチャ

![regrailのアーキテクチャ](docs/architecture.svg)

正規表現はまず再帰下降パーサでASTに変換します。各ノードには一意のidを振り、鉄道図の描画とマッチトレースのハイライトを同じidで結びつけます。照合は継続渡しのバックトラッキングで行い、ステップを逐次記録します。UIはこれらの結果を描画するだけの薄い層で、パーサ・照合・描画はいずれもDOM非依存です。

## プロジェクト構成

- `src/ast.ts` — ノードの型定義
- `src/parser.ts` — 正規表現をASTに変換する再帰下降パーサ
- `src/charset.ts` — 文字クラス・短縮クラスの判定
- `src/matcher.ts` — バックトラッキング照合エンジンとステップ記録
- `src/railroad.ts` — ASTを鉄道図SVGにレイアウトする
- `src/main.ts` — 図の描画とステップ実行のUI
- `src/examples.ts` — 厳選したサンプルパターン
- `src/state.ts` — 入力内容をURLハッシュに保存・復元する
- `src/theme.ts` — ライト・ダーク・自動の切り替えと永続化
- `src/icons.ts` — UIで使うモダンSVGアイコン
- `docs/architecture.svg` — アーキテクチャ図

## 設計方針

- **ASTのidで図と照合を接続** — 描画ノードとトレースを同じidで対応づけ、ステップ実行のハイライトを成立させる
- **正しさはネイティブと突合** — 自前エンジンの一致範囲・捕捉を `RegExp` と比較するテストで担保する
- **過程を記録できる照合** — 結果だけでなく試行・消費・後戻りをトレースとして残し、可視化の素材にする
- **状態をURLに持たせる** — 正規表現とテスト文字列をURLハッシュに保存し、リンクひとつで再現・共有できるようにする
- **依存ゼロ** — パーサも照合も描画も自前で持つ

## 制約

後方参照、先読み・後読み、名前付きグループ、フラグ(`i` `g` `m` など)には対応していません。照合は複数行を意識しない単純な動作で、`.` は改行に一致しません。教育・デバッグ用途に向けた標準的な記法のサブセットを対象としています。
